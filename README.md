### Изначальная формулировка задания


Телеграм-бот с поддержкой множественных диалогов (с разными пользователями).

———————————————————————————

Команды:
+ /start - начало работы с ботом
+ /info - информация о боте и командах
+ /task - шахматная задача
+ /stats - статистика пользователя

Как будет выглядеть получение задачи
По команде от пользователя бот пришлёт ему задачу в подобном формате:

⚔️ Мат в 4 хода
⬜️ Играем за белых

✊🏿👌🏿☝🏿👩🏿🤴🏿☝🏿👌🏿✊🏿
👃🏿👃🏿👃🏿👃🏿👃🏿👃🏿👃🏿👃🏿
⬜️⬛️⬜️⬛️⬜️⬛️⬜️⬛️
⬛️⬜️⬛️⬜️⬛️⬜️⬛️⬜️
⬜️⬛️⬜️⬛️⬜️⬛️⬜️⬛️
⬛️⬜️⬛️⬜️⬛️⬜️⬛️⬜️
👃🏻👃🏻👃🏻👃🏻👃🏻👃🏻👃🏻👃🏻
✊🏻👌🏻☝🏻👩🏼🤴🏻👌🏻☝🏻🤙🏻✊🏻

После чего пользователь вводит ход в формате стандартной английской шахматной нотации (Nd3, e7, Kh1 ...)
Если ход правильный, задача продолжается, бот присылает позицию с ответным ходом, и так продолжается, пока задача не решена.
Если ход неправильный или введённые данные не те, бот сообщает об этом, и ждёт нового ответа.

Дополнительные возможности:
+ задачи на время
+ количество попыток
+ разные уровни сложности задач, с возможностью выбора
+ разные типы задач (мат, выигрыш фигуры)

———————————————————————————

Что будет в статистике
Пользователь увидит количество решённых задач разного уровня сложности

———————————————————————————

Данные будут храниться в облачной дб Mongo.

Пользователь:
+ id
+ решенные задачи (список id задач)

Задача: 
+ id
+ название (опционально, просто есть знаменитые задачки с названиями)
+ тип задачи
+ количество ходов
+ сложность
+ изначальная позиция и очередь хода (в специальном шахматном формате FEN (https://en.wikipedia.org/wiki/Forsyth%E2%80%93Edwards_Notation))
+ решение (список ходов: "пешка, e5-e6"...)

———————————————————————————

GUI:
+ разметка кнопок в телеграмме, выбор команды и выбор сложности при команде task
+ генерация позиций эмодзи (наверное считается)
